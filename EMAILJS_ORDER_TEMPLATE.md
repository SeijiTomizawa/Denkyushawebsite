### 1. テンプレートを保存

EmailJSのテンプレート編集画面で **Save** ボタンをクリック

### 2. 開発サーバーを再起動

```bash
# サーバーを停止（Ctrl+C）して再起動
npm run dev
```

### 3. ブラウザで確認

ブラウザのコンソール（F12）を開いて、以下のログを確認：

## 🔄 2つのフォームの区別

現在、以下のように完全に分離されています：

### お問い合わせフォーム
- **テンプレートID:** `template_vffjunb`
- **変数:** `from_name`, `from_email`, `company`, `message`
- **用途:** 簡易的な問い合わせ

### お申し込みフォーム
- **テンプレートID:** `template_p95e9ze`（新規作成）
- **変数:** `company_name`, `person_name`, `email`, `phone`, など14個
- **用途:** 詳細なWEB制作依頼

それぞれ独立したテンプレートなので、**片方を変更してももう片方に影響しません**。

## 📞 トラブルシューティング

### Q: まだ古いテンプレートが使われている

**A:** 以下を確認：
1. 開発サーバーを再起動したか
2. ブラウザのキャッシュをクリアしたか
3. コンソールログで `orderTemplateId: "template_p95e9ze"` と表示されているか

### Q: メールは届くが内容が空白

**A:** EmailJSのテンプレートで変数名を確認：
- ❌ `{{companyName}}` （キャメルケース）
- ✅ `{{company_name}}` （アンダースコア）

変数名は完全に一致する必要があります。

### Q: お問い合わせフォームも影響を受けた

**A:** 影響を受けません。お問い合わせフォームは `template_vffjunb` を使い続けます。
コンソールで確認できます：
- お問い合わせ: `templateId: "template_vffjunb"`
- お申し込み: `templateId: "template_p95e9ze"`
